# BusTracker (Minsktrans)

Проект показывает на OLED-дисплее **приближение автобусов** к выбранной остановке в Минске: данные берутся с [minsktrans.by](https://www.minsktrans.by), упрощаются на **Python-прокси** (из-за TLS, парсинга и тяжёлого JSON), а **ESP8266** только запрачивает компактный JSON по HTTP в локальной сети.

## Архитектура

| Компонент | Назначение |
|-----------|------------|
| **`src/minsktrans.py`** | Асинхронный клиент к официальному фронту/API: загрузка страницы, CSRF-токен, «антибот»-преобразование параметров из JS, ограничение частоты запросов (~3 req/s). |
| **`src/proxy.py`** | HTTP-сервер (aiohttp): собирает табло остановки, трек маршрута и позиции ТС, считает расстояние до остановки по полилинии и отдаёт JSON на `GET /buses`. |
| **`src/main.cpp`** | Прошивка для **ESP8266** (профиль `esp01_1m`), дисплей **SSD1306** 128×64 по I²C, опрос прокси раз в ~20 с. |

Поток данных: **minsktrans.by → Python-прокси → ESP8266 → OLED**.

## Требования

- **Прокси:** Python 3.10+ (в Docker — 3.11, см. `src/Dockerfile`).
- **Прошивка:** [PlatformIO](https://platformio.org/), плата **ESP8266** в конфиге — `esp01_1m` (1 MB flash), фреймворк Arduino.
- **Библиотеки (PIO):** Adafruit GFX, Adafruit SSD1306 (указаны в `platformio.ini`).

Зависимости Python: `src/requirements.txt` (`aiohttp`, `beautifulsoup4`).

## Установка и запуск прокси

Из каталога `src/`:

```bash
python -m venv .venv
.venv\Scripts\activate   # Windows
pip install -r requirements.txt
python proxy.py
```

Сервер слушает порт **8080** (см. `PROXY_PORT` в `proxy.py`). Эндпоинт: `http://<IP_вашего_ПК>:8080/buses`.

### Docker

```bash
cd src
docker build -t minsktrans-proxy .
docker run -p 8080:8080 minsktrans-proxy
```

## Настройка прокси (`proxy.py`)

В начале файла задаются:

- **`STOP_ID`** — ID остановки на minsktrans (по умолчанию в репозитории — пример; **замените на свой**).
- **`TARGET_ROUTE`** — номер нужного маршрута строкой (как на табло), например `"56"`.
- **`AVG_SPEED_MPS`** — средняя скорость для грубой оценки ETA по расстоянию (м/с).

Ответ `/buses` — JSON с полями вроде `stop`, `buses` (до 4 ближайших к остановке по треку), `official` (минуты с табло, если доступны).

### Где взять номер маршрута и ID остановки

1. Откройте интерактивную карту Minsktrans: [Lookout (Минск)](https://www.minsktrans.by/lookout_yard/Home/Index/minsk).
2. **Номер маршрута** — это тот же номер, что на остановке и в списке маршрутов на сайте (автобус, троллейбус, трамвай). В коде он задаётся в **`TARGET_ROUTE`** без лишних символов, например `56`, `29`, `100` (тип транспорта в `proxy.py` должен совпадать с тем, что вы выбрали: для автобуса используется `TransportType.Bus`, как в текущем примере).
3. **ID остановки (`STOP_ID`)** на публичной странице часто не подписан явно. Надёжный способ:
   - на карте выберите нужную остановку так, чтобы загрузилось табло с этой остановки;
   - откройте инструменты разработчика браузера (F12) → вкладка **Сеть / Network**;
   - обновите данные или кликните по остановке ещё раз;
   - найдите запрос к API вида `.../Data/Scoreboard` (POST) и посмотрите тело или параметры запроса: там будет поле остановки (часто **`s`** — это и есть значение для **`STOP_ID`** в `proxy.py`).

Если после смены остановки или маршрута на карте появляется другой URL или параметры — ориентируйтесь на тот `s=...`, который уходит в запросе табло для **вашей** остановки.

## Прошивка ESP8266 и сеть

### Свой Wi‑Fi и доступ к прокси

Используйте **свою** домашнюю (или тестовую) сеть — **не** чужие учётные данные из примеров в коде.

- **`WIFI_SSID` / `WIFI_PASSWORD`** в `src/main.cpp` — имя и пароль **вашей** точки доступа, к которой подключается модуль.
- **ESP8266 работает только в диапазоне 2,4 ГГц.** Сеть «только 5 GHz» для него недоступна; включите на роутере совместимый 2,4 GHz SSID или отдельную гостевую сеть 2,4 GHz.
- **`PROXY_HOST`** — **локальный IP компьютера**, на котором запущен `proxy.py` (тот же Wi‑Fi, что и у ESP). Узнать: в Windows — `ipconfig` (IPv4-адрес адаптера Wi‑Fi); адрес не должен меняться при каждом включении ПК (при необходимости задайте резервирование IP в настройках роутера).
- **`PROXY_PORT`** — должен совпадать с `PROXY_PORT` в `proxy.py` (по умолчанию **8080**).
- ПК с прокси и ESP должны видеть друг друга в одной LAN; брандмауэр Windows может блокировать входящие на порт 8080 — при отсутствии ответа разрешите Python или порт **8080** для частной сети.

Телефон в режиме точки доступа подойдёт, если и компьютер с прокси, и ESP подключены к этой же точке и вы прописали IP ноутбука в `PROXY_HOST`.

### Сборка и заливка

1. Заполните **`WIFI_SSID`**, **`WIFI_PASSWORD`**, **`PROXY_HOST`**, **`PROXY_PORT`** своими значениями.
2. Соберите и залейте прошивку через PlatformIO (`pio run -t upload` или из IDE).

**Важно:** не коммитьте в git реальные пароли Wi‑Fi и внутренние IP; держите их в локальных правках или в конфиге вне репозитория.

## Ограничения и заметки

- Клиент в `minsktrans.py` отключает проверку SSL-сертификата сайта (`CERT_NONE`) — это осознанный обход проблем с цепочкой/сертификатом у источника; для прокси на доверенной машине это обычно приемлемо, но не копируйте такой режим в публичные сервисы без оценки рисков.
- ESP8266 использует минимальный разбор JSON строками — формат ответа прокси не меняйте без правок прошивки.
- Память ESP-01 1M ограничена; `platformio.ini` включает профиль низкой памяти LWIP.
